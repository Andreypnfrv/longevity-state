# Policy commitment — фиксированные требования и направления

## Типы первичных доказательств (общий словарь)

Использовать **глубокие ссылки** в `links`; в `date` — год/период документа или бюджетного года.


| Код   | Что считается доказательством                                                                                                                                                                                             | Не считается                                         |
| ----- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------- |
| **L** | Закон, план-закон, постановление правительства / решение совета министров с **именованной** программой или обязанностью органа                                                                                            | Пресс-релиз без нормативного текста                  |
| **B** | Строка/пакет в **официальном** бюджете, казначействе, consolidated appropriations, госпрограмме с **явной суммой** и горизонтом (финансовый год или многолетка)                                                           | Общие обещания «увеличим финансирование» без таблицы |
| **A** | Устав/положение **агентства или постоянного** органа с мандатом (в т.ч. внутри министерства) + атрибуция бюджета в том же контуре                                                                                         | Временная рабочая группа без бюджета                 |
| **C** | **Непрерывность**: тот же мандат/линия в документе **после** смены правительства / нового плана / нового коалиционного договора (аналог для не-демократий — новый пятилетний/плановый цикл с сохранением главы программы) | Одна волна без продления                             |


Критерий «что именно коммитим» (только geroscience vs включая уход/профилактику) — **одно правило на весь набор стран**; в narrative страницы явно писать, какой объект оценки.

## Модель A — жёсткие гейты по рунгу (один балл)

Ниже «И» = логическое **И** внутри рунга, если не сказано иначе. Балл = **максимальный** рунг, чьи условия выполнены полностью.

### Рунг 0

- **Либо** **L⁻**: нормативный акт **блокирует** longevity-relevant исследования без обзорного пути  
- **Либо** нет **ни одного** из **L/B/A** по **научному** мандату на старение/геронтологию (только соцуход/пенсии без R&D) — и это зафиксировано первичкой

*(Если в продукте сплитнуть 0: два подтипа — obstruction vs care-only-no-science.)*

### Рунг 1

- Есть **L** или официальная стратегия высшего исполнительного уровня по здоровью/старению **но** нет **B** с выделенной строкой/суммой под именованную программу старения/долголетия (только общий «health» / «science» пул без разбивки)

### Рунг 2

- **L** с именованной программой **и** **B** с суммой и горизонтом **≤ одного ожидаемого политического цикла** (или явно годовая линия без многолетней гарантии в первичке)  
- **И** нет **C** (не доказана пережившая смену власти/плана)

### Рунг 3

- **L** + **B** (многолетка **≥ 3 года** в официальном документе **или** последовательные годовые акты с той же линией)  
- **И** **C** (одна задокументированная «волна» непрерывности после смены кабинета/плана/коалиции **или** эквивалент для плановой экономики)  
- **И** межведомственный след: **≥ 2 министерства** в одном L/B пакете (или явное совместное письмо/соглашение в primary)

### Рунг 4

- Всё из рунга **3** **плюс** **одно** из:  
  - отдельное **A** с устойчивым мандатом («агентство / институт / нац. центр») и отдельной бюджетной атрибуцией;  
  - **или** «флагман» в **L** с формулировкой **explicit** healthy longevity / aging biology / equivalent (не только «dementia» без aging frame — см. общее правило объекта оценки)
- **И** бюджетный масштаб в narrative квалифицируется как **крупный относительно пиров** (без жёсткого % в схеме — в тексте страницы 1–2 сравнения или ссылка на OECD/World Bank R&D)

### Рунг 5

- **L** уровня план-закона / эквивалента с **количественной** целью по здоровью населения (LE, DFLE, «healthy life years» — как в документе) **и** процедурой пересмотра  
- **И** **B** с защищённой базой (многолетка или индексация/«базовая линия» в первичке)  
- **И** **C** через **≥ 2** независимых политических/плановых цикла **или** эквивалентная норма «отмена только высоким решением»

## Модель B — баллы за направления (закон/план + бюджет)

**Направление засчитывается**, только если одновременно:

1. **L** (или равносильный нормативный акт) с **именованным** блоком под это направление
2. **B** с **выделенной** суммой или субсидией под тот же блок (или однозначная ссылка из L на приложение с цифрами)

Максимум **4 направления** (чтобы не раздувать счётчиком дубликатов одного и того же акта):


| #   | Направление (примерный ярлык)         | Содержание                                                               |
| --- | ------------------------------------- | ------------------------------------------------------------------------ |
| D1  | **Aging biology / geroscience R&D**   | Фундаментальная биология старения, геронтология, не только одна болезнь  |
| D2  | **Diseases of aging / translation**   | Деменция, онко/метаболика возраста, клинические исследования, трансляция |
| D3  | **Care / LTCI / dependency**          | Страхование ухода, система зависимости, институциональный уход           |
| D4  | **Prevention / public health ageing** | Нац. скрининги, профилактика, healthy ageing в первичке                  |


**Маппинг в существующую шкалу 0–5:**


| Набрано направлений (каждое = L+B) | Условие непрерывности **C**                      | Итоговый балл (предложение)                        |
| ---------------------------------- | ------------------------------------------------ | -------------------------------------------------- |
| 0 и нет L по старению              | —                                                | **0–1** (0 если obstruction / только уход без L+B) |
| 1                                  | нет                                              | **2**                                              |
| 1                                  | да                                               | **3**                                              |
| 2                                  | да                                               | **3**                                              |
| 3                                  | да                                               | **4**                                              |
| 4                                  | да + явный количественный якорь здоровья в **L** | **5**                                              |


При **конфликте** моделей A и B — в продукте выбрать одну; рекомендация: **модель A как источник истины**, модель B как **внутренний чеклист** в комментариях к оценке.

## Следующие шаги (когда решите внедрять)

1. Зафиксировать в [claims-and-links.mdc](.cursor/rules/claims-and-links.mdc) ссылку на этот план и требование L/B/A/C для `POLICY_COMMITMENT`.
2. При необходимости подправить формулировки рунгов в [src/schema/index.ts](src/schema/index.ts) под выбранную модель.
3. Пройти страны и выровнять `links` под коды доказательств (в `comment` можно помечать `L+B`, `C`).
